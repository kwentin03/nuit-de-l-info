# Guide de contribution

## Avant de commencer
1. Consultez le README.md
2. Comprenez les trois piliers: Inclusion, Responsabilité, Durabilité

## Processus de contribution
1. Créer une branche: git checkout -b feature/nom-feature
2. Effectuer vos changements
3. Commit avec messages explicites
4. Push et créer une pull request

## Règles de commit
- Messages en français ou anglais, clairs et concis
- Préfixe: feat:, fix:, docs:, refactor:, test:
